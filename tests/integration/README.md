# Circle Guard — Integration Tests

Tests that validate **real communication between microservices** using TestContainers,
RestAssured, and EmbeddedKafka. These tests run against actual PostgreSQL, Redis, and
Kafka instances (spun up via Docker) — no H2, no mocks for infrastructure.

---

## Quick Start

```bash
# Make the script executable (once)
chmod +x tests/integration/run_integration_tests.sh

# Run all integration tests (from repo root)
./tests/integration/run_integration_tests.sh

# Run a specific test class
./tests/integration/run_integration_tests.sh --test AuthGatewayIntegrationTest

# Or use Gradle directly
./gradlew :tests:integration:test --info
```

## What Each Test Validates

### 1. `AuthIdentityIntegrationTest` — AUTH ↔ IDENTITY (HTTP Synchronous)

| Test | What it proves |
|------|---------------|
| `loginWithValidCredentials_returnsJwtToken` | Auth service generates a valid JWT on successful login |
| `loginWithWrongPassword_returns401` | Auth service rejects bad credentials |
| `loginSuccess_callsIdentityService` | Auth service makes a **real HTTP call** to identity service (WireMock intercepts) |
| `mapIdentity_withoutJwt_returns401` | Identity service requires JWT authentication |
| `lookupIdentity_withoutJwt_returns401` | Lookup endpoint requires JWT |
| `mapIdentity_withJwtMissingPermission_returns403` | Identity service enforces role-based access control |

**Infrastructure used:** PostgreSQL (auth DB), Embedded LDAP, WireMock (identity mock on port 8083)

---

### 2. `AuthGatewayIntegrationTest` — AUTH ↔ GATEWAY (QR Validation Flow)

| Test | What it proves |
|------|---------------|
| `validateQr_withValidToken_gateOpens` | Valid QR JWT (same secret as auth) opens the gate |
| `validateQr_withExpiredToken_gateRejects` | Expired QR is rejected |
| `validateQr_withTamperedToken_gateRejects` | Tampered QR signature is rejected |
| `validateQr_withMissingToken_returns4xx` | Missing token returns error |
| `validateQr_calledTwice_bothSucceed` | Second call served from Redis cache |

**Infrastructure used:** Redis (GenericContainer redis:7-alpine)

**Key integration point:** QR tokens generated by auth service use `QR_SECRET`. Gateway validates
using the same secret. Test generates tokens inline with JJWT to prove the contract works.

---

### 3. `PromotionNotificationKafkaTest` — PROMOTION → NOTIFICATION (Kafka Async)

Promotion service is **not started** (avoids Neo4j dependency). Messages are published
directly to Kafka topics, simulating what promotion-service would emit.

| Test | What it proves |
|------|---------------|
| `publishStatusChanged_notificationListenerProcessesEvent` | `ExposureNotificationListener` consumes `promotion.status.changed` |
| `publishConfirmedStatus_triggersAlertProcessing` | CONFIRMED status triggers alert logic |
| `publishPriorityAlert_confirmedCase_listenerProcesses` | `PriorityAlertListener` consumes `alert.priority` with CONFIRMED_CASE |
| `publishPriorityAlert_largeOutbreak_listenerProcesses` | LARGE_OUTBREAK (15 affected) triggers priority alert |
| `publishCircleFenced_listenerProcesses` | `CircleFencedListener` consumes `circle.fenced` (triggers room cancellation) |
| `publishMultipleEvents_allConsumed` | 5 concurrent events — none dropped |

**Infrastructure used:** EmbeddedKafka (topics: promotion.status.changed, alert.priority, circle.fenced)

---

### 4. `FormNotificationKafkaTest` — FORM → NOTIFICATION (Kafka Async)

| Test | What it proves |
|------|---------------|
| `submitSurveyWithSymptoms_publishesSurveySubmittedEvent` | Submitting survey with fever+cough publishes `survey.submitted` |
| `submitSurveyWithoutSymptoms_publishesNegativeEvent` | Symptom-free survey still publishes event (hasSymptoms=false) |
| `validateCertificate_publishesCertificateValidatedEvent` | Approving a certificate publishes `certificate.validated` |
| `getPendingCertificates_returnsList` | Pending certificates list endpoint is functional |

**Infrastructure used:** PostgreSQL (form DB), EmbeddedKafka

**Key integration point:** A test `@KafkaListener` captures messages from both topics, allowing
assertion on the exact Kafka payload produced by the form service.

---

### 5. `CircleEncounterFlowTest` — E2E Multi-Service Pipeline

Two nested test suites sharing EmbeddedKafka as the message bus:

**Phase 1 (Form Service):**
| Test | What it proves |
|------|---------------|
| `studentSubmitsHealthSurvey_eventPublishedToKafka` | Full round-trip: REST → DB → Kafka |
| `multipleStudentsSubmitSurveys_allEventsReachKafka` | Concurrent submissions without message loss |

**Phase 2 (Notification Service):**
| Test | What it proves |
|------|---------------|
| `notificationService_consumesStatusChangedEvent` | SUSPECT status processed by ExposureNotificationListener |
| `notificationService_consumesCircleFencedEvent` | circle.fenced processed by CircleFencedListener |
| `notificationService_consumesPriorityAlert_confirmedOutbreak` | CONFIRMED_CASE processed by PriorityAlertListener |
| `fullPipeline_suspectToConfirmedToFenced_allEventsProcessed` | **Full 4-event sequence**: SUSPECT → CONFIRMED → priority alert → circle fenced |


## Prerequisites

- **Docker Desktop** running (TestContainers needs it)
- **Java 21**
- **Gradle wrapper** (already present at repo root)
- First run pulls Docker images (~500 MB total): `postgres:15`, `redis:7-alpine`

## Timeouts

Each test has `@Timeout(60)` — 60 seconds max per test. Full suite runs in ~3-5 minutes.
